# 學習日誌：階段四 - 打造專屬風格與客製化設定
**日期：2026-02-26**

今天順利完成了網站「階段四」的客製化設定！在這個階段中，我學習了如何透過修改 `_config.yml` 與各項靜態資源，讓這個技術部落格更貼近我個人的風格與調性。

## 🌟 今日完成事項

### 1. 變更網站主色調 (Color Scheme)
我將網站的主題切換為了深色模式 (Dark Mode)。這不僅符合許多開發者的閱讀習慣，視覺上也讓版面看起來更有質感。學習到了只要在 `_config.yml` 裡面調整 `color_scheme: dark` 就能夠快速地全站套用。

### 2. 首頁視覺升級：加入專屬 Banner
我替首頁 (`index.md`) 加上了一張自製的 Banner 圖片（`assets/images/banner.png`）。在 Markdown 中插入圖片非常直覺，這樣做也讓首頁的 `Welcome` 區塊看起來不再只有單調的文字，變得更加吸睛。

### 3. 設定網頁分頁圖示 (Favicon)
以前總是好奇瀏覽器分頁上的小圖示是怎麼加上去的，原來這叫做 Favicon！我把專屬的圖示 (`assets/images/favicon.png`) 放進專案資料夾後，也在 `_config.yml` 中透過 `favicon_ico` 參數成功綁定了這個圖示。

### 4. 客製化頁尾版權與宣告 (Footer)
學習到可以透過設定 `footer_content` 參數，覆寫原本主題自帶的網頁底部資訊。我加上了版權聲明，以及一句我非常喜歡的話來展現個人風格：
> 「完美是古老鐘擺的蘚苔，唯有那帶裂痕的、過度張揚的熵增，才是靈魂呼吸時吞吐出的代碼塵埃。」

## 💡 學習體悟
今天的操作大多專注在 `_config.yml` 的參數調整以及圖片資源的管理。這讓我深刻體會到靜態網站產生器 (如 Jekyll 或 Hexo) 強大且靈活的地方：我們不需要從零開始刻畫 HTML/CSS，只要善用主題提供的設定檔，就能大幅改變網站的外觀與各種細節。
