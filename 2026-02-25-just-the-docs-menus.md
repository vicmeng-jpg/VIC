# 📝 學習筆記：Just the Docs 的選單魔法

這篇筆記統整了我在設定網站分類選單時學習到的重要觀念。
透過這些設定，我可以輕鬆地把文章分門別類，讓整個部落格看起來更有條理！

## 1. 核心觀念：YAML Front Matter
每篇 Markdown 文章最上方由 `---` 包住的區塊，是專門寫給系統看的「設定檔」。系統就是靠讀取裡面的關鍵字來決定網頁長什麼樣子的。

## 2. 關於排版 (Layout)
- `layout: default`：預設的一般文章版型，因為閱讀區塊較為集中，通常旁邊會有幫助跳轉段落的大綱目錄。
- `layout: home`：通常用給首頁或是大目錄頁面，排版上會更專注在中央內容。

## 3. 建立側邊欄多層次選單的步驟

要在左側拉出分類資料夾，其實只要兩步：

### Step A: 建立一個「主目錄頁 (父節點)」
建立一個新的 `.md` 檔案，在 YAML 中加入：
```yaml
nav_order: 3        # 決定這個主目錄排在第幾個
has_children: true  # 宣告「我底下有包含其他文章」
```

### Step B: 把「文章 (子節點)」塞進去
在想放進該目錄的文章 YAML 中加入：
```yaml
parent: 這裡寫主目錄的標題名稱  # 要跟父節點的 title 完全一樣！
```

> **進階技巧：第三層選單！**  
> 如果想建立「主目錄 -> 子目錄 -> 文章」的三層結構，只要在中層和底層的文章分別善用 `parent` 和 `grand_parent` 就可以達成了！

## 4. 排序的藝術 (nav_order)
`nav_order` 決定了文章從上到下的排列順序，**數字越小排越上面**。
實務上的小技巧是：如果希望**「最新的文章永遠在最上面」**，我們可以用「負數的日期」來當作 `nav_order` (例如 `-20260225`)，這樣隨著日期越來越大，它加上負號後就會越來越小，自然就自動置頂了！
